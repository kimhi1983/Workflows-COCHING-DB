# FORMULATION SKILL — 처방 설계

> COCHING AI 가이드처방 파이프라인 Step 1/6
> 제품타입 + 피부타입 기반 투입 원료 선정 및 wt% 배분

---

## 1. 트리거 조건
- 사용자가 가이드 처방 생성 요청 시 최초 실행
- `/guide-formula` 슬래시 커맨드 호출 시 자동 시작

## 2. 입력
- 제품타입 (예: 선크림, 세럼, 로션 등)
- 피부타입 (건성/지성/복합성/민감성/중성)
- 특수 요구사항 (SPF, 무향, 비건, 특정 성분 포함/제외 등)

## 3. 처리 절차

### Step 1 — 기존 처방 조회
```
DB 조회: guide_cache WHERE combo_key = '{제품타입}_{피부타입}'
  → 기존 처방 존재 시: 참조 처방으로 활용
  → 미존재 시: 신규 처방 설계
```

### Step 2 — 처방 구조 설계
```
Phase 분류 (화장품 제조 표준):
  A상 (수상): 정제수 + 수용성 원료 (보습제, 점증제, 킬레이트제)
  B상 (유상): 오일 + 유화제 + 에몰리언트
  C상 (기능성): 자외선 차단제, 색소 등 (제품별)
  D상 (첨가제): 활성성분, 방부제, 향료 (저온 투입)
```

### Step 3 — 투입 원료 선정 기준
```
1. 피부타입별 원료 적합성
   - 민감성: 무기자차, 저자극 유화제, 무향, 진정 성분
   - 지성: 경량 오일, 세바시안 컨트롤, 논코메도제닉
   - 건성: 고보습, 세라마이드, 오클루시브
   - 복합성: T존/U존 밸런스, 경-중 오일 혼합
   - 중성: 범용 배합

2. 규제 한도 확인
   - 자외선 차단제: 한국 식약처 배합 한도 준수
   - 방부제: 배합 한도 (예: Phenoxyethanol ≤ 1.0%)
   - 색소: 허용 색소 목록 확인

3. 원료 호환성
   - 유화 시스템 (O/W, W/O, W/S) 선택
   - pH 호환성 (활성성분 안정성 pH 범위)
   - 원료 간 상호작용 (비호환 조합 회피)
```

### Step 4 — wt% 배분
```
밸런스 원리:
  정제수(Aqua) = 100% - (기타 모든 성분 합계)
  → 밸런스는 최종 역산으로 결정 (PRECISION-ARITHMETIC에서 처리)

배분 가이드라인:
  유효 성분: 제품 목적에 따라 적정 농도
  유화제: 3-5% (유화 시스템에 따라)
  보습제: 5-15% (피부타입에 따라)
  오일/에몰리언트: 5-20% (제형에 따라)
  자외선 차단제: SPF 목표에 따라 (SPF50: ZnO 15% + TiO2 7% 등)
  방부제: 규제 한도 이내
  활성성분: 유효 농도 이상
  정제수: 밸런스 (통상 30-70%)
```

## 4. 출력
- Phase별 투입 원료 목록 (한글명, INCI명, wt%, 기능)
- 각 원료의 유형 분류 (SINGLE / COMPOUND / BALANCE)
- 제조 온도 조건
