### 1-2. DB 스키마 변경

```sql
-- guide_cache_copy.estimated_ph: numeric → text
-- Gemini가 "N/A (무수 시스템)" 등 텍스트를 반환하므로 text 타입으로 변경
```

> **MyLab 영향**: guide_cache_copy 테이블에서 estimated_ph를 숫자로 파싱하던 코드가 있다면 text 처리로 변경 필요

## 4. MyLab-Studio 연동 시 주의사항

### 4-1. API 쿼리 변경 사항

```sql
-- 원료의 5개국 규제 조회 (신규 데이터 활용)
SELECT rc.inci_name, rc.restriction::jsonb->>'country' as country,
       rc.restriction::jsonb->>'status' as status,
       rc.restriction::jsonb->>'ewg_score' as ewg_score,
       rc.restriction::jsonb->>'primary_function' as function
FROM regulation_cache rc
WHERE rc.source = 'GEMINI_SAFETY'
  AND rc.inci_name = '조회할 INCI명';

-- US FDA 최신 규제 조회
SELECT rc.ingredient, rc.restriction::jsonb->>'summary' as summary,
       rc.restriction::jsonb->>'severity' as severity,
       rc.restriction::jsonb->>'url' as source_url
FROM regulation_cache rc
WHERE rc.source = 'REG_MONITOR_US'
ORDER BY rc.updated_at DESC;
```

### 4-2. guide_cache_copy.estimated_ph 처리

```javascript
// [이전] 숫자로 파싱
```
