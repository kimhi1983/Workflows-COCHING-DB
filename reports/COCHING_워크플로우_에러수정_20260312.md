## 2. 수정 내용

### FIX 1: Gemini API 키 교체
- **대상**: 원료 수집 v7, 제품 수집 v1
- **원인**: `AIzaSyAXhVn...` 키가 leaked로 차단
- **수정**: 새 API 키 `AIzaSyBBFQlk...`로 교체
- **스크립트**: `scripts/fix_workflows.py`

### FIX 2: 제품카피 SQL 파라미터 바인딩
- **대상**: 제품카피 가이드 v1.0 → `💾 카피가이드 저장` 노드
- **원인**: `'{{ $json.guide_data_escaped }}'::jsonb` 방식으로 JSON 직접 삽입 → 작은따옴표 이스케이프 실패
- **수정**: PostgreSQL 파라미터 바인딩 ($1~$9) 방식으로 완전 교체

```
변경 전: VALUES ('{{ $json.source }}', ... '{{ $json.guide_data_escaped }}'::jsonb, ...)
변경 후: VALUES ($1, $2, $3, $4, $5::jsonb, $6, $7, $8, $9, NOW())
         queryReplacement: ={{ [$json.source_product_id, ...] }}
```

- **스크립트**: `scripts/fix_copy_sql_v2.py`

### FIX 3: Track 1 비활성화
- **대상**: v2.3-A~F (6개 워크플로우)
- **원인**: 215/215 전체 완료 → 처리 대상 없음 → 매 2분마다 에러 발생
- **수정**: `active=0`, `triggerCount=0`, `activeVersionId` 갱신
- **교훈**: n8n에서 `active=0`만으로 비활성화 안 됨 → activeVersionId + pm2 restart 필요

### FIX 4: coching-ai PM2 삭제
- **이유**: Anthropic API 토큰 과금 → Max 플랜 CLI로 대체 가능
- **방식**: `pm2 delete coching-ai` + `pm2 save`
- **파일 보존**: `Coching-Cosmetic-AI-V1/ai-server/` 참조용 유지
