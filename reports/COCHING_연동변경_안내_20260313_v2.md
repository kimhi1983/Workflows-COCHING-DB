## 마이랩에서 반드시 수정해야 할 항목 (3건)

### 1. `estimated_ph` 타입 변경 (numeric → text)

**DB 변경**: `guide_cache_copy.estimated_ph` 컬럼이 `numeric`에서 `text`로 변경됨

```javascript
// ❌ 이전 — 숫자 파싱
const ph = parseFloat(row.estimated_ph);

// ✅ 현재 — 텍스트 그대로 표시
// Gemini가 "N/A", "5.5-6.0", "N/A (무수 시스템)" 등을 반환
const ph = row.estimated_ph; // string 처리
// 숫자 표시가 필요하면:
const phNum = parseFloat(row.estimated_ph);
const phDisplay = isNaN(phNum) ? row.estimated_ph : phNum.toFixed(1);
```

**영향 범위**: guide_cache_copy 테이블에서 estimated_ph를 `parseFloat`, `Number()`, `toFixed()` 등으로 처리하는 모든 곳

### 2. regulation_cache 신규 소스 2종 연동

기존 `coching_legacy`, `GEMINI_KR`, `GEMINI_EU` 외에 **2개 소스가 추가**됨:

| 신규 소스 | 건수 | 구조 | 용도 |
|---|---|---|---|
| `GEMINI_SAFETY` | **325건** (증가 중) | restriction.country별 1행 | 원료별 5개국 규제+안전성 |
| `REG_MONITOR_US` | **54건** (증가 중) | 문서 1건 = 1행 | US FDA 최신 규제 문서 |

#### GEMINI_SAFETY 쿼리 (원료별 5개국 규제)

```sql
-- 특정 원료의 5개국 규제 조회
SELECT
  rc.inci_name,
  rc.restriction::jsonb->>'country' AS country,
  rc.restriction::jsonb->>'status' AS status,
  rc.restriction::jsonb->>'ewg_score' AS ewg_score,
  rc.restriction::jsonb->>'primary_function' AS primary_function,
  rc.restriction::jsonb->>'concerns' AS concerns,
  rc.restriction::jsonb->>'cir_assessment' AS cir_assessment,
  rc.restriction::jsonb->>'note' AS note
FROM regulation_cache rc
WHERE rc.source = 'GEMINI_SAFETY'
  AND rc.inci_name ILIKE '%Niacinamide%';
```

**restriction JSON 구조**:
```json
{
  "country": "KR",           // KR, EU, US, JP, CN
  "status": "allowed",       // allowed, restricted, banned
  "annex": "III",            // EU Annex 번호 (해당 시)
  "max_concentration": "5%", // 최대 허용 농도
  "ewg_score": 1,            // EWG 점수 (1~10)
  "concerns": ["접촉 알레르기 가능성"],
  "primary_function": "피부 컨디셔닝제",
  "cir_assessment": "안전한 것으로 평가",
  "note": "화장품 원료로 자유롭게 사용 가능"
}
```

#### REG_MONITOR_US 쿼리 (US FDA 최신 규제)

```sql
-- US FDA 최신 규제 문서 조회
SELECT
  rc.ingredient,
  rc.restriction::jsonb->>'summary' AS summary,
  rc.restriction::jsonb->>'severity' AS severity,   -- low, medium, high
  rc.restriction::jsonb->>'category' AS category,    -- rule, guidance, notice
  rc.restriction::jsonb->>'url' AS source_url,
  rc.restriction::jsonb->>'published_at' AS published_at
FROM regulation_cache rc
WHERE rc.source = 'REG_MONITOR_US'
ORDER BY rc.updated_at DESC
LIMIT 20;
```

**restriction JSON 구조**:
```json
{
  "region": "US",
  "category": "guidance",           // rule, guidance, notice
  "severity": "medium",             // low, medium, high
  "summary": "자외선 차단제 성분에 대한 FDA 안전성 평가 가이던스 발행",
  "url": "https://www.federalregister.gov/...",
  "keywords": ["sunscreen", "UV filter"],
  "published_at": "2026-03-10"
}
```
